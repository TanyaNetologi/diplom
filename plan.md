2. Используемые инструменты и обоснование выбора
Python 3.12 - Основной язык разработки автотестов.

Pytest - Фреймворк для организации и запуска тестов. Поддерживает параметризацию, фикстуры и удобное формирование отчётов.

Selenium WebDriver - Инструмент для автоматизации пользовательского интерфейса веб-приложения.

webdriver-manager - Автоматическое управление драйверами браузеров без ручной установки.

Psycopg2 - Библиотека для подключения к PostgreSQL и проверки данных в базе после выполнения операций.

Requests - Библиотека для работы с HTTP-запросами и возможной проверки API.

Allure Pytest - Инструмент формирования отчётов о выполнении автотестов.

Docker и Docker Compose - Средство развёртывания тестового окружения (приложение, БД, платёжный шлюз).

Git и GitHub - Хранение исходного кода, документации, отчётов и ведение задач через Issues.

3. Возможные риски при автоматизации
Настройка взаимодействия контейнеров Docker Compose.
Ошибки подключения приложения к PostgreSQL.
Задержка записи данных в БД после выполнения операции.
Нестабильность UI-тестов из-за времени загрузки страниц.
Возможные отличия реализации примеров диплома от Python-стека.
Дополнительные затраты времени на настройку Allure и интеграцию отчётов.
Изменения структуры данных в БД во время выполнения проекта.
